# 数据库

> 版本：v0.1.11 | 更新日期：2026-07-11

## 数据库总览

所有数据库统一存放在 `{DATA_PATH}/Reference/`（默认 `D:\Quotation_Automation\Reference\`）。其中 5 个由 `base.py` 定义独立的 engine + sessionmaker，`comments.db` 由 `comment_repo.py` 以原生 `sqlite3` 直接管理。各库相互独立，严禁混用。

| 数据库 | 用途 | 管理方式 |
|--------|------|----------|
| `kp_data.db` | KP 零件价格、分类数据；业务字段（business_fields）及其引用、审计、使用统计 | `base.py` engine |
| `l6_data.db` | L6 整机型号、五维匹配记录 | `base.py` engine |
| `opportunities.db` | 商机线索、报价单、报价配置项、商机文件 | `base.py` engine |
| `rules.db` | 业务规则（区域识别、KP 分类映射、主板映射、匹配规则）、导出模板 | `base.py` engine |
| `l6_history.db` | L6 价格变更历史快照 | `base.py` engine |
| `comments.db` | 商机批注 | 原生 sqlite3（`comment_repo.py`） |

Engine / SessionLocal 定义位置：`backend/app/models/base.py`

> 说明：每张表实际写入哪个库，由其 Repository 绑定的 SessionLocal 决定。下文各表的归属均以代码中的 Repository 绑定为准。

## 表结构明细

### 1. kp_data.db

#### kp_records — KP 零件价格

| 字段 | 类型 | 说明 |
|------|------|------|
| `rowid` | INTEGER PK | 自增主键（无显式 PK，使用 rowid） |
| `category` | TEXT | 分类（如 CPU、内存、硬盘） |
| `model` | TEXT | 型号 |
| `price` | REAL | 最新价格 |
| `currency` | TEXT | 币种（默认 RMB） |
| `date` | TEXT | 更新日期 |
| `note` | TEXT | 备注 |

> Repository：`repository/kp_repo.py`（`KPRepository`，使用 `kp_engine`，以原生 SQL 访问）。

#### business_fields — 业务字段配置

业务字段（Business Field）动态管理模块的核心配置表，定义了可在商机/报价单/整机/零件等数据源上挂载的动态字段。`PricingEngine` 在导出时会读取此表动态解析字段值。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK | 自增主键 |
| `key` | TEXT UNIQUE | 字段标识（唯一） |
| `label` | TEXT | 字段显示名 |
| `category` | TEXT | 归类：`opportunity` / `item` / `l6` / `kp` / `system` |
| `source` | TEXT | 数据源模型：`Opportunity` / `L6Record` / `KPRecord` / `System` |
| `source_column` | TEXT | 数据源对应列名 |
| `type` | TEXT | 数据类型：`text` / `number` / `date` / `boolean`（默认 `text`） |
| `enabled` | BOOLEAN | 是否启用（默认 `True`） |
| `sort_order` | INTEGER | 排序权重（默认 0） |
| `description` | TEXT | 字段描述（v2） |
| `display_type` | TEXT | 展示类型：`text` / `money` / `percent` / `date` / `enum`（默认 `text`，v2） |
| `group_name` | TEXT | 分组名（v2） |
| `scope` | TEXT | 作用域：`all` / `cover` / `config` / `rules`（默认 `all`，v2） |
| `permission` | TEXT | 权限：`editable` / `readonly` / `hidden`（默认 `editable`，v2） |
| `validation_rules` | TEXT | 校验规则（JSON：`required`/`pattern`/`min`/`max`/`minLength`/`maxLength`，v3） |
| `options` | TEXT | 枚举可选项（JSON 数组，v3） |
| `dependencies` | TEXT | 字段依赖关系（JSON：`field`/`operator`/`value`，v3） |
| `created_at` | TEXT | 创建时间 |
| `updated_at` | TEXT | 更新时间 |
| `created_by` | TEXT | 创建者（默认 `system`） |
| `updated_by` | TEXT | 更新者（默认 `system`） |

> Repository：`repository/business_field_repo.py`（`BusinessFieldRepository`，使用 `KP_SessionLocal`）。

#### field_references — 字段引用关系

记录某个业务字段被哪些模板/导出/规则引用，删除前用于引用检查。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK | 自增主键 |
| `field_key` | TEXT | 关联 `business_fields.key` |
| `ref_type` | TEXT | 引用类型：`template` / `export` / `rule` |
| `ref_id` | INTEGER | 引用方 ID |
| `ref_name` | TEXT | 引用方名称 |
| `created_at` | TEXT | 创建时间 |

#### field_audit_logs — 字段变更审计

记录业务字段的创建/更新/删除操作及变更内容。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK | 自增主键 |
| `field_key` | TEXT | 关联字段 |
| `action` | TEXT | 操作类型：`create` / `update` / `delete` |
| `changes` | TEXT | 变更明细（JSON：`{"字段": {"old": ..., "new": ...}}`） |
| `operator` | TEXT | 操作人（默认 `system`） |
| `operated_at` | TEXT | 操作时间 |

#### field_usage_stats — 字段使用统计

统计每个业务字段被使用次数及最近使用时间。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK | 自增主键 |
| `field_key` | TEXT UNIQUE | 关联字段（唯一） |
| `usage_count` | INTEGER | 累计使用次数（默认 0） |
| `last_used_at` | TEXT | 最近使用时间 |

## 建表机制

- `rules.db` 与 `l6_history.db` 的表由 `core/startup.py` 的 `init_rules_db()` 在应用启动时通过 `Base.metadata.create_all()` 创建并初始化默认规则。
- `kp_data.db` / `l6_data.db` / `opportunities.db` 为既有业务数据库，由各自 Repository 直接读写。
- `comments.db` 由 `comment_repo.py` 按需创建。
