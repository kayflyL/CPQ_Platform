# 配件页（BasePricing）优化方案

> 基于对 `BasePricing.vue`（1153 行）、`backend/app/repository/kp_repo.py`、`backend/app/models/kp.py`、`frontend/src/styles/tokens.css` 的调研。
> 目标：视觉与功能一起升级，沿用现有 Cyberpunk Dark 青色主题；修复已知 bug；补齐层级分类、关联配件、批量导入导出。

---

## 一、现状诊断

### 1.1 功能正确性问题（明确的 bug）

| 问题 | 现状 | 后端 | 前端 |
|---|---|---|---|
| 排序下拉是死控件 | `sortBy` 绑了 `@change` 但请求未带参 | `list_parts` 写死 `order_by(name)`，无排序入参 | `loadParts` 未传 `sortBy` |
| 卡片视图无分页 | `pagination` 仅在 `a-table` 生效 | 正常 | 卡片视图无翻页入口，>20 条看不见 |
| 分类编辑用原生 prompt | 风格割裂 | 正常 | `editCategory` 用 `window.prompt` |
| 报价日期手打 | 易格式错 | 正常 | `a-input` 文本框，应为 `a-date-picker` |

### 1.2 视觉不一致

整套设计系统的强调色是**亮青色 `#00F5D4`**（`--cpq-accent-primary`），但配件页存在多处偏离：

- 左侧分类选中态用了蓝色 `rgba(24,144,255,0.1)`（第 757 行），与主题青色冲突。
- "成功色"定义本身有冲突：`--cpq-accent-success: #00F5D4`（青）与 `--cpq-color-success: #52c41a`（绿）并存；卡片价格用青、价格趋势图却用绿 `#52c41a`，同一类信息两种颜色。
- 卡片 hover 用硬编码 `box-shadow: 0 2px 8px rgba(0,0,0,0.1)`，而 tokens 里已有 `--cpq-shadow-sm/md/lg`。
- 图表配色（`#52c41a`、`rgba(82,196,26,0.1)`、`#888` 等）全部硬编码，未走变量。

### 1.3 被埋没的后端能力（数据模型有、前端没用）

`KPPart` 模型里 `full_desc`、`image_url`、`datasheet_url`、`moq`(起订量) 均未展示；`KPCategory` 的 `parent_id`(层级)、`icon`、`sort_order` 未利用；`KPPriceHistory` 的 `source`(来源)、`currency` 未录入；更有一整张 `KPPartRelated` 关联配件表及对应接口（`admin.py` 287–310 行）前端完全未接入。

---

## 二、设计原则

1. **颜色只用 `tokens.css` 变量**（项目"硬编码零容忍"原则），含图表色集中定义为 JS 常量。
2. **不推翻现有暗色青色风格**，做层次与一致性强化，而非换皮。
3. 后端改动遵循 **API → Service → Repository** 分层；Engine 不碰 DB。
4. 所有路径基于 `Path(__file__).parent` 动态获取，不硬编码。

---

## 三、P0 — 必修 Bug（功能正确性）

最小改动、最快见效，建议第一批合入。

**P0-1 排序失效修复**
- `kp_repo.list_parts`：新增 `sort_by`、`sort_order` 参数，支持 `name` / `updated_at` / `price`(asc|desc)。价格排序需 join 最新报价（子查询取 `MAX(price_date)` 的价格）。
- `admin.list_parts`：透传两个参数。
- 前端 `loadParts`：把 `sortBy`（如 `name-asc`）解析后随请求发出。
- 注意：价格排序的子查询要 `nullslast()`，避免无报价配件乱序。

**P0-2 卡片视图补分页**
- 卡片网格下方加 `a-pagination`，复用现有 `pagination` state 与 `handleTableChange` 逻辑；视图切换时保留当前页。

**P0-3 分类编辑去 prompt**
- 把 `editCategory` 的 `window.prompt` 改为复用一个内联编辑 modal（可在现有"分类管理"弹窗内做行内编辑态），与暗色主题一致。

**P0-4 报价日期选择器**
- "新增报价"modal 中日期字段 `a-input` → `a-date-picker`，值格式 `YYYY-MM-DD`。

## 四、P1 — 视觉升级（一致性 + 层次）

**P1-1 颜色统一**
- sidebar 选中态：`rgba(24,144,255,0.1)` → `var(--cpq-overlay-a10)`，文字色用 `var(--cpq-accent-primary)`。
- 价格色统一为青色系：卡片价格、表格价格、详情价格列表、趋势图全部用同一图表常量（建议 `--cpq-accent-primary` #00F5D4，搭配 `overlay-a10` 填充）。
- 替换所有硬编码 `rgba(0,0,0,…)` 阴影为 `var(--cpq-shadow-*)`；`#888` 等灰阶替换为 `--cpq-text-muted`。
- 图表色在 `<script>` 顶部集中定义为常量对象（因 chart.js 读不到 CSS 变量），便于维护。

**P1-2 卡片重设计**
- hover：边框切 `var(--cpq-accent-primary)` + `var(--cpq-shadow-md)`；顶部加一条 2px 青色 accent 条（默认透明，hover 显现）。
- 信息块重组：品牌用 `a-tag`、成色用彩色徽章（全新=青、翻新=橙、拆机=灰）、货期用小图标 + 文字。
- "暂无报价"卡片整体降饱和（opacity 0.7），与有价卡片形成层次。

**P1-3 详情抽屉增强**
- 抽屉头部：名称 + 品牌徽章 + 分类 tag 组合，替代纯标题。
- 各 section 间距、分隔线统一；价格趋势图容器高度与配色调整。
- "兼容机型"区由纯 tag 升级为可点击（未来可跳转机型页）。

**P1-4 表单与状态收敛**
- loading 收敛：卡片视图用骨架屏（`a-skeleton`）或独立 spin，避免与表格自带 loading 重叠闪烁。
- 空状态增加"新增第一个配件"引导按钮；搜索无结果时区分"无数据"与"未命中"。

## 五、P2 — 功能增强（盘活后端已有能力）

### P2-1 层级分类树

后端 `KPCategory` 已有 `parent_id`、`icon`、`sort_order`，能力齐备。

- 后端 `list_all_categories`：返回结构已含 `parent_id`；可选新增一个返回树形 JSON 的端点，或前端自行组装。
- 前端 sidebar：平铺列表 → 树形（`a-tree` 或递归列表），支持展开/折叠；子分类计数聚合到父级。
- "分类管理"modal：新增"父分类"下拉（选"无"为顶级），并支持设置 `icon` 与 `sort_order`。
- 迁移注意：现有平铺分类 `parent_id` 为 null，自动作为顶级，无需数据迁移。

### P2-2 关联配件推荐

后端接口已存在（`GET/POST/DELETE /kp/parts/{id}/related`，admin.py 287–310），前端未接。

- 详情抽屉新增"关联配件"区块：调用 `GET related`，以小卡片展示（名称 + SKU + 最新价），点击可跳转对应配件详情。
- 编辑表单新增"关联配件"多选（带搜索的其他配件），保存调 `POST/DELETE related`。
- 落地前先抓一次 `GET related` 确认返回字段结构，再定渲染。

### P2-3 批量导入 / 导出

项目本身是 Excel 报价工具，openpyxl / pandas / exceljs 基础设施齐全。

- **导出**：工具栏加"导出"按钮。为绕开分页、保证全量，建议后端新增 `GET /api/admin/kp/parts/export`（按当前筛选条件，流式返回 xlsx），前端触发下载。列含：名称/分类/SKU/品牌/成色/最新价/更新日。
- **导入**：工具栏加"导入"按钮 → 上传 xlsx → 后端新增 `POST /api/admin/kp/parts/import`，解析后批量 `create_part`。需做：必填 `name` 校验、`oem_sku` 去重、逐行错误回报（返回成功数 + 失败行明细）。
- **模板下载**：提供标准导入模板（含表头说明与示例行），降低填错率。

## 七、实施顺序与验证

建议 **P0 → P1 → P2** 依次合入，每期可独立验证、独立回滚。

验证要点：
- 改后端后用 `python -m py_compile` 校验，防止 VM 文件截断（项目已知坑）。
- 颜色改动后 `grep` 检查无硬编码 `#xxxxxx` / `rgba(` 残留。
- 层级分类、导入功能若涉及建表/迁移，留意"首次建库"流程。

---

## 八、风险与注意事项

- 价格排序需 join 子查询，注意 SQLite 下的 SQL 性能（配件量大时可对 `part_id` + `price_date` 建索引）。
- 批量导入时 SQLite 写锁，建议单事务批量提交并在导入前备份库文件。
- 关联配件、层级分类均依赖已有后端接口，落地前务必抓一次真实返回确认字段，避免按假设渲染。
