# 动态字段管理架构实施方案

## 背景

### 问题
原有实现采用硬编码白名单方式管理字段，新增字段需要修改多处代码：
- 数据库 Schema
- ORM 模型
- Repository 层
- API 层
- 前端组件

### 目标
实现"零代码改动"新增字段能力：只需在字段系统配置新字段，系统自动支持存储、读取、展示。

### 方案
**混合存储 + 配置驱动**：
- 核心字段（高频查询、业务关键）→ 独立列
- 扩展字段（业务扩展、低频查询）→ `extra_fields JSON`
- API 和前端动态读取字段系统配置

### 多租户预留
当前不引入多租户，但底层设计预留 `tenant_id` 字段（默认 'default'），为未来多租户平滑过渡做准备。

## 三阶段实施计划

### 第一阶段：Opportunities 层架构验证 ✅ 已完成

**目标**：验证动态字段架构在商机层的可行性，打通全链路。

#### 改造内容

**1. 数据库 Schema**
- `opportunities` 表：新增 `extra_fields TEXT` + `tenant_id VARCHAR DEFAULT 'default'`

**2. ORM 模型**
- `Opportunity` (`models/opportunity.py`)：
  - 新增 `extra_fields` 和 `tenant_id` 字段
  - `to_dict()` 自动展开 `extra_fields` 到顶层

**3. Repository 层**
- `OpportunityRepository` (`repository/opportunity_repo.py`)：
  - 定义 `_CORE_COLUMNS`（17个核心字段）
  - `update_meta()` 方法：核心字段→列，其他→`extra_fields JSON`

**4. API 层**
- `OpportunityUpdate` (`api/opportunities.py`)：
  - 添加 `model_config = {"extra": "allow"}` 支持动态字段

**5. Engine 层**
- `pricing_engine.py`：
  - `get_opportunity_details()` 使用 `dict(project_dict)` 动态展开所有字段
  - `_get_meta_value()` 支持动态字段解析

**6. 前端**
- `OpportunityDetail.vue`：
  - 使用 `v-for="field in dynamicFields"` 动态渲染表单项
  - 数据加载使用 `{...meta}` 展开所有字段

#### 验证结果
- **6/6 PASS**：Save core+dynamic, Read & expand, DB routing, Preserve existing, Empty string, Fields API

#### 架构效果
- 新增商机字段（如 `industry`、`project_source`）只需在字段系统配置
- 无需修改后端代码、前端代码
- 自动支持存储、读取、展示

#### 实施步骤

**Step 1: 多租户依赖注入**
```python
# backend/app/dependencies.py
def get_current_tenant(request: Request) -> str:
    """Extract tenant_id from request header or session."""
    tenant_id = request.headers.get("X-Tenant-ID", "default")
    return tenant_id

# backend/app/repository/opportunity_repo.py
def list_opportunities(self, tenant_id: str = "default") -> List[Opportunity]:
    return self.session.query(Opportunity).filter(
        Opportunity.tenant_id == tenant_id,
        Opportunity.status == "active"
    ).all()
```

**Step 2: 字段系统权限控制**

```sql
CREATE INDEX idx_business_fields_tenant ON business_fields(tenant_id);
```
